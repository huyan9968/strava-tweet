### 1.2 获取应用凭证

创建成功后，您将看到以下信息（⚠️ **请立即复制保存**）：

```

Client Secret:    [您的 Client Secret] ⬅️ 保存这个
```

### 1.3 获取 OAuth Refresh Token

Strava 使用 OAuth 2.0 进行认证，我们需要获取 Refresh Token。

#### 方法 A：使用浏览器获取（推荐）

1. 打开浏览器，访问以下链接（替换 YOUR_CLIENT_ID）：

```
https://www.strava.com/oauth/authorize?client_id=YOUR_CLIENT_ID&response_type=code&redirect_uri=https://localhost&approval_prompt=force&scope=read_all,activity:read_all
```

2. 授权后，浏览器会跳转到：
```
https://localhost/?code=AUTHORIZATION_CODE
```

3. 复制 `code` 参数值（AUTHORIZATION_CODE）

4. 使用 curl 获取 Refresh Token：

```bash
curl -X POST https://www.strava.com/oauth/token \
  -d client_id=YOUR_CLIENT_ID \
  -d client_secret=YOUR_CLIENT_SECRET \
  -d code=AUTHORIZATION_CODE \
  -d grant_type=authorization_code
```

5. 响应中会包含：
```json
{
  "access_token": "xxx",
  "athlete": {...},
  "refresh_token": "YOUR_REFRESH_TOKEN",  ⬅️ 保存这个
  "expires_at": 1234567890,
  "expires_in": 21600
}
```

#### 方法 B：使用 Python 脚本（更简单）

```python
import webbrowser

client_id = "YOUR_CLIENT_ID"
url = f"https://www.strava.com/oauth/authorize?client_id={client_id}&response_type=code&redirect_uri=https://localhost&approval_prompt=force&scope=read_all,activity:read_all"

print("正在打开浏览器...")
print(f"请访问: {url}")
webbrowser.open(url)

auth_code = input("\n请输入授权码 (code 参数): ")

import requests

response = requests.post('https://www.strava.com/oauth/token', data={
    'client_id': client_id,
    'client_secret': 'YOUR_CLIENT_SECRET',
    'code': auth_code,
    'grant_type': 'authorization_code'
})

token_data = response.json()
print(f"\n✅ Refresh Token: {token_data['refresh_token']}")
print(f"⚠️  请妥善保存，不要分享给他人！")
```

## 第二步：部署 Cloudflare Worker

### 2.1 创建 Cloudflare 账户

1. 访问：https://dash.cloudflare.com/
2. 注册/登录 Cloudflare 账户

### 2.2 创建 Worker

1. 在左侧菜单找到 **"Workers & Pages"**
2. 点击 **"Create Application"** → **"Worker"**
3. 点击 **"Create"**

### 2.3 部署代码

将以下代码粘贴到编辑器中：

```javascript
/**
 * Strava Webhook → GitHub Actions 触发器
 * 
 * 环境变量（在 Cloudflare 后台设置）：
 *   STRAVA_VERIFY_TOKEN  自定义验证字符串
 *   GITHUB_TOKEN         GitHub PAT（需要 repo 权限）
 */

const GITHUB_REPO = 'huyan9968/strava-tweet';

export default {
  async fetch(request, env) {
    const url = new URL(request.url);

    // ── GET：Strava 注册 Webhook 时的验证请求 ──────────
    if (request.method === 'GET') {
      const mode      = url.searchParams.get('hub.mode');
      const challenge = url.searchParams.get('hub.challenge');
      const token     = url.searchParams.get('hub.verify_token');

      if (mode === 'subscribe' && token === env.STRAVA_VERIFY_TOKEN) {
        return Response.json({ 'hub.challenge': challenge });
      }
      return new Response('Forbidden', { status: 403 });
    }

    // ── POST：Strava 推送新活动通知 ───────────────────
    if (request.method === 'POST') {
      let body;
      try {
        body = await request.json();
      } catch {
        return new Response('Bad Request', { status: 400 });
      }

      // 只处理新活动创建事件
      if (body.object_type === 'activity' && body.aspect_type === 'create') {
        const resp = await fetch(
          `https://api.github.com/repos/${GITHUB_REPO}/dispatches`,
          {
            method: 'POST',
            headers: {
              Authorization:  `token ${env.GITHUB_TOKEN}`,
              Accept:         'application/vnd.github.v3+json',
              'Content-Type': 'application/json',
              'User-Agent':   'strava-webhook-worker',
            },
            body: JSON.stringify({ event_type: 'strava_activity' }),
          }
        );

        if (!resp.ok) {
          const text = await resp.text();
          return new Response(`GitHub trigger failed: ${text}`, { status: 502 });
        }
      }

      return new Response('OK');
    }

    return new Response('Not Found', { status: 404 });
  },
};
```

4. 点击 **"Save and Deploy"** 按钮
5. 复制 Worker 的 URL（类似：https://your-worker.your-name.workers.dev/）

### 2.4 配置环境变量

1. 在 Worker 页面，点击 **"Settings"** 标签
2. 找到 **"Variables"** 部分
3. 点击 **"Add"** 添加以下变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `STRAVA_VERIFY_TOKEN` | `your-secret-token` | 自定义字符串（如："my-strava-webhook-2024"） |
| `GITHUB_TOKEN` | `ghp_xxxxxxxxxxxx` | GitHub Personal Access Token |

4. 点击 **"Save"** 保存

## 第三步：在 Strava 注册 Webhook

### 3.1 获取必要信息

确保您已有：
- ✅ Strava App 的 `Client ID`
- ✅ Strava App 的 `Client Secret`
- ✅ OAuth `Refresh Token`
- ✅ Cloudflare Worker 的 URL
- ✅ 自定义的 `Verify Token`

### 3.2 注册 Webhook

使用 curl 命令注册（替换相应参数）：

```bash
curl -X POST https://www.strava.com/api/v3/push_subscriptions \
  -d client_id=YOUR_CLIENT_ID \
  -d client_secret=YOUR_CLIENT_SECRET \
  -d 'callback_url=https://your-worker.your-name.workers.dev/' \
  -d 'verify_token=your-secret-token'
```

成功的响应：
```json
{
  "id": 12345,
  "resource_state": 2,
  "object_type": "activity",
  "aspect_type": "create",
  "mode": "exercise",
  "callback_url": "https://your-worker.your-name.workers.dev/",
  "verify_token": "your-secret-token",
  "application_id": 12345
}
```

### 3.3 验证 Webhook

Strava 会发送 GET 请求验证：
- Worker 接收验证请求
- 检查 `hub.verify_token`
- 返回 `hub.challenge`

如果配置正确，Webhook 状态会显示为 **"Active"** ✅

## 🚨 常见问题

### Q1: Strava Webhook 验证失败
**解决方案**：
- 检查 Cloudflare Worker 的 `STRAVA_VERIFY_TOKEN`
- 确保与注册时的 Verify Token 一致
- Worker URL 不要有拼写错误

### Q2: GitHub Actions 权限不足
**解决方案**：
- 确认 GITHUB_TOKEN 有 `repo` 权限
- 重新生成 Personal Access Token

### Q3: OAuth 授权码过期
**解决方案**：
- Refresh Token 长期有效
- 如果失效，重新执行 OAuth 流程

### Q4: 网站没有更新
**解决方案**：
- 检查 GitHub Actions 是否执行成功
- 确认 GitHub Pages 分支设置正确
- 等待几分钟（GitHub Pages 构建需要时间）

### Q5: 地图生成失败
**解决方案**：
- 确认 routes/ 目录存在
- 检查是否有写权限
- 确保 polyline 数据有效
